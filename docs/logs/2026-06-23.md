## 6. Security error response 수정

진행 브랜치:

```text
fix/security-error-response
```

문제:

- 중복 이메일이나 validation 오류가 원래 `409`, `400`이 아니라 `403`으로 내려오는 현상 발생

원인:

- Spring Boot 예외 처리의 `/error` dispatch가 Spring Security에 막힘

작업 내용:

- `DispatcherType.ERROR` 허용
- `/error` 경로 허용
- 문제 원인과 수정 내용을 문서화

생성/수정 파일:

```text
docs/common/01-security-error-response.md
src/main/java/com/petopscommerce/global/config/SecurityConfig.java
```
