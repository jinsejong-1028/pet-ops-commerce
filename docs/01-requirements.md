# 요구사항 정의

## 사용자 역할

| 역할 | 설명 |
|---|---|
| Guest | 상품 조회와 회원가입 가능 |
| Member | 장바구니, 주문, 반려동물 프로필 관리 가능 |
| Operator | 주문 상태 확인, 재고 확인, 고객 응대 가능 |
| Admin | 상품, 쿠폰, 재고, 운영 설정 관리 가능 |

## 기능 요구사항

### 회원/인증

- 사용자는 이메일과 비밀번호로 가입할 수 있다.
- 사용자는 로그인 후 JWT를 발급받는다.
- 관리자는 사용자 권한을 변경할 수 있다.
- API는 권한에 따라 접근을 제한한다.

### 반려동물 프로필

- 회원은 반려동물 이름, 종류, 생년월일, 알레르기 정보를 등록할 수 있다.
- 추천/운영 기능은 반려동물 프로필을 참고할 수 있다.

### 상품

- 관리자는 상품을 등록, 수정, 판매 중지할 수 있다.
- 사용자는 상품 목록과 상세 정보를 조회할 수 있다.
- 상품은 카테고리, 가격, 판매 상태, 재고 상태를 가진다.

### 장바구니

- 회원은 상품을 장바구니에 담을 수 있다.
- 회원은 장바구니 수량을 변경하거나 삭제할 수 있다.
- 장바구니 조회 시 판매 중지 상품과 재고 부족 상품을 구분한다.

### 주문

- 회원은 장바구니 기반으로 주문을 생성할 수 있다.
- 주문 생성 시 재고를 확인한다.
- 주문 상태는 생성, 결제대기, 결제완료, 출고준비, 출고완료, 취소로 관리한다.
- 주문 취소 시 재고를 복구한다.

### 재고

- 관리자는 창고별 상품 재고를 조회할 수 있다.
- 입고, 출고, 이동, 조정 이력을 남긴다.
- 동시 주문 상황에서도 재고가 음수가 되지 않아야 한다.

### 쿠폰

- 관리자는 쿠폰을 생성할 수 있다.
- 회원은 주문 시 쿠폰을 적용할 수 있다.
- 쿠폰은 기간, 최소 주문 금액, 중복 사용 가능 여부를 가진다.

### 운영 로그/알림

- 주요 도메인 이벤트를 저장한다.
- 재고 부족, 주문 실패, 결제 실패 이벤트를 운영자가 조회할 수 있다.
- 추후 Slack, Email, Webhook 알림으로 확장한다.

### AI 운영 보조

- 운영자는 주문/문의/상품 데이터를 기반으로 고객 응대 초안을 생성할 수 있다.
- 초기 구현은 외부 LLM API를 직접 호출하지 않고 Mock 인터페이스로 시작한다.
- 이후 Python FastAPI 서비스 또는 OpenAI API 연동으로 확장한다.

## 비기능 요구사항

- API 서버는 stateless하게 설계한다.
- 인증 상태는 JWT로 관리한다.
- 공유 상태는 DB, Redis, Message Queue에 둔다.
- 다중 API 서버 환경에서도 동일하게 동작해야 한다.
- 주문/재고 핵심 로직은 테스트 코드로 보호한다.
- 장애 추적을 위해 요청 ID와 주요 이벤트 로그를 남긴다.
- 주요 조회 API는 페이징과 검색 조건을 지원한다.
- DB 인덱스와 QueryDSL을 통해 검색 성능을 개선한다.
